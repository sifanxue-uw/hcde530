# Week 2 — Midterm reflection

## Competency 2: Code literacy and documentation

### What this competency means to me

To me, code literacy is less about memorizing syntax and more about an attitude or mindset: being willing to engage with code even when it feels unfamiliar, and writing in a way that other people can understand what you are doing—not only machines.

### What I did this week (evidence)

This week I intentionally documented my work in several ways. I added `#` comments in the code to explain what each section does. I used descriptive commit messages (for example, notes like “added comments” and “new files”) to track my progress in version control. I also chose meaningful variable names so the code would be easier to read at a glance.

### How I used documentation (comments, README, naming, etc.)

I relied on inline comments, clear naming, and commit messages as my main documentation. Section-level `#` comments are becoming my default so I do not have to re-read the whole file from scratch later.

### What I learned about reading or writing code (with or without tools)

I cannot usually read code straight through and understand it yet. Instead I go step by step: I ask AI to explain each part, and I run the code to see what it outputs. Because I have little prior coding experience, almost everything still feels new, but that loop—explain, run, observe—helps me gradually make sense of it. I used Cursor and AI extensively; they helped me generate code, explain pieces, and debug errors. I have not had to fix much on my own so far unless something looked obviously wrong. I still rely heavily on AI, but I am learning to read program output and check whether results look correct.

### One concrete example (file, snippet, or decision)

I plan to add `#` comments in both English and Chinese in `analyze.py`. Bilingual comments should help me understand the code better while keeping it readable for others who prefer either language.

### What I want to improve next

Before the end of the course, I want to get better at guiding AI so I get the output I actually need, instead of accepting the first thing it generates. Stronger prompting and clearer goals should make my workflow more intentional and my results more reliable.
